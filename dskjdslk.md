**ВВЕДЕНИЕ**

Создание приложения для инвентаризации обувного магазина на языке C# с использованием технологий ADO.NET является востребованной задачей в рамках цифровизации малого и среднего бизнеса. Современные методы доступа к данным, в частности асинхронное взаимодействие с SQL Server через `SqlConnection` и параметризованные запросы, обеспечивают надёжное хранение информации и высокую скорость обработки запросов. Разработка такого продукта помогает оптимизировать складской учёт, минимизировать ошибки, связанные с человеческим фактором, и повысить общую эффективность управления торговым объектом.

**Цель практики:**  
Научиться писать программный код на языке C# с использованием платформы .NET и технологии ADO.NET для работы с внешними источниками данных. Освоить механизмы взаимодействия веб-приложения (Blazor Server) с базой данных SQL Server. Уметь формировать алгоритмы для разработки программных модулей в соответствии с техническим заданием. Собрать практический опыт для выполнения последующих учебных и дипломных работ.

**Задачи практики:**
- Закрепить знания синтаксиса C# и принципов объектно-ориентированного программирования при создании модели данных `Shoe`.
- Освоить работу с библиотеками `Microsoft.Data.SqlClient`: создание подключений, параметризованных команд, построчное чтение данных через `SqlDataReader` и агрегацию через `ExecuteScalar`.
- Научиться формировать алгоритмы обработки данных, включая асинхронное добавление записей, фильтрацию и пересчёт показателей.
- Изучить возможности платформы Blazor Server для построения интерактивного веб-интерфейса с динамическим обновлением UI.
- Продемонстрировать работу готовой программы и составить отчётную документацию по результатам практики.

**Объект исследования:** Учебная практика проходила в Автономной некоммерческой организации высшего образования «Санкт-Петербургский институт экономики и управления» (АНО ВО «СПбИЭУ»).

**Предмет исследования:** Написание кода на языке C# с использованием технологии Blazor Server и ADO.NET для создания приложения «Обувной салон».

**1 РЕЗУЛЬТАТЫ ПРОХОЖДЕНИЯ ПРАКТИКИ**

В данном разделе представлены итоги работы над программным модулем, предназначенным для автоматизации учёта товаров в обувном ритейле.

**1.1 Инструменты разработки и их возможности**

Для реализации проекта использовался следующий стек инструментов, доступный на операционной системе Linux (Fedora):

1. **Среда выполнения .NET 8** – обеспечивает работу приложения Blazor Server и компиляцию кода C#.
2. **Редактор кода (Visual Studio Code / nano)** – применялся для написания и редактирования исходных файлов. Поддерживает подсветку синтаксиса, что ускоряет разработку.
3. **Командная строка .NET CLI** – использовалась для создания проекта (`dotnet new blazor`), добавления пакетов (`Microsoft.Data.SqlClient`) и публикации готового приложения.
4. **SQL Server (локально)** – развёрнут на том же хосте, где велась разработка. Для управления базой данных применялся графический клиент DBeaver, позволявший выполнять скрипты создания таблиц и просматривать данные.
5. **Браузер** – служил средой выполнения пользовательского интерфейса (Blazor Server рендерит страницы и передаёт их через SignalR).

Такая конфигурация позволила полностью отказаться от проприетарной среды Windows и продемонстрировать кроссплатформенность современных .NET-технологий.

**1.2 Создание программы «Обувной салон» на языке C#**

Процесс разработки включал следующие этапы:
- **Выбор типа проекта.** Был создан проект Blazor Server с интерактивным рендерингом (`interactivity Server`). Это обеспечило выполнение всего C#-кода на стороне сервера и мгновенное обновление UI через веб-сокеты.
- **Проектирование базы данных.** В SQL Server создана таблица `Shoes` с полями: `Id` (первичный ключ), `Model`, `Brand`, `Size`, `Price`. Скрипт создания таблицы и тестовых записей оформлен в виде отдельного SQL-файла.
- **Настройка подключения.** Класс `DatabaseHelper` содержит статический метод `GetConnection()`, возвращающий открытое соединение `SqlConnection` с использованием безопасной строки подключения.
- **Реализация бизнес-логики:**
  - Класс `ShoeRepository` инкапсулирует всю работу с БД. Метод `AddShoeAsync` принимает объект `Shoe` и выполняет параметризованную команду `INSERT`, гарантируя защиту от SQL-инъекций.
  - Чтение списка товаров осуществляется через `SqlDataReader` в методе `GetAllShoesAsync`, что обеспечивает потоковую обработку и высокую производительность.
  - Методы `GetTotalCountAsync` и `GetTotalSumAsync` используют `ExecuteScalar` для получения агрегированных данных (общее количество пар и их суммарная стоимость).
  - Фильтрация реализована динамическим построением SQL-запроса с добавлением условий только на те поля, которые указаны пользователем, при этом все значения передаются через параметры.
- **Интерфейс пользователя.** Компонент `Home.razor` содержит форму добавления, панель фильтров, таблицу со списком обуви и информационную строку с итоговыми показателями. Благодаря использованию `@rendermode InteractiveServer` все события (клики, ввод) обрабатываются на сервере, а UI обновляется автоматически.

**1.3 Структура данных и тестовые записи для базы данных**

Для проверки корректности работы системы в таблицу `Shoes` были добавлены начальные записи:

| Модель       | Бренд | Размер | Цена    |
|--------------|-------|--------|---------|
| Air Max      | Nike  | 42.0   | 9999.99 |
| Old Skool    | Vans  | 40.5   | 7999.50 |

В дальнейшем через интерфейс приложения можно добавлять произвольные модели, указывая бренд, размер и цену. Все записи сразу отображаются в таблице, а агрегированные показатели пересчитываются.

**ЗАКЛЮЧЕНИЕ**

За время прохождения учебной практики был разработан полнофункциональный программный модуль «Обувной салон» на языке C# с использованием технологий Blazor Server и ADO.NET.

**Основные итоги:**
- Закреплены навыки объектно-ориентированного программирования: создана модель данных `Shoe`, реализован репозиторий для работы с SQL Server.
- Освоена технология ADO.NET: настроено подключение через `SqlConnection`, применены параметризованные запросы для безопасного взаимодействия с БД.
- Реализована потоковая выборка данных (`SqlDataReader`) и агрегирование (`ExecuteScalar`), что обеспечило высокую производительность приложения.

**Предложения по улучшению:**
